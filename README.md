# GP5 — Deep Learning для платформы секонд-хенда

Групповой проект №5 по курсу «Современные методы анализа данных и машинного обучения».
НИУ ВШЭ, 2025-26 учебный год.

## Бизнес-контекст

Проект выполнен в рамках задач платформы по продаже одежды секонд-хенд.
Две связанные бизнес-задачи:

- **Задача 1 (изображения):** по фотографии вещи предсказать её состояние (`condition`, шкала 1–5)
- **Задача 2 (таблица):** по характеристикам товара, включая предсказанное состояние, предсказать цену

Предсказанный признак `condition` из первой задачи добавляется в табличный датасет и используется как фича во второй задаче — таким образом обе задачи образуют единый пайплайн.

## Модели

### Задача 1 — предсказание состояния по фото (ординальная регрессия)

Сравниваются три архитектуры с pretrained весами (ImageNet):

- ResNet-50
- EfficientNet-B3
- ViT-B/16

Целевая переменная — `condition` (1–5), ординальная. Используется CORN loss.
Метрики: Accuracy, MAE, Macro F1.

### Задача 2 — предсказание цены (MLP на табличных данных)

Сравниваются три архитектуры полносвязных сетей:

- Base — 3 слоя (128 -> 64 -> 1), ReLU, без регуляризации, 11k параметров
- BN — 4 слоя (256 -> 128 -> 64 -> 1), BatchNorm, 47k параметров
- BN_Dropout — 5 слоёв (512 -> 256 -> 128 -> 64 -> 1), BatchNorm + Dropout(0.3), 186k параметров

Целевая переменная — price (SEK), непрерывная. Таргет логарифмирован (log1p). Используется MSE loss. Метрики: MAE (sek), RMSE (sek), R².
Дополнительно проведён подбор learning rate для лучшей модели (BN).

## Логирование экспериментов

Эксперименты логируются с помощью MLflow. Каждая архитектура — отдельный run с параметрами, метриками по эпохам и артефактом модели.

Папка `mlruns/` добавлена в `.gitignore`. Для воспроизведения экспериментов необходимо запустить обучение локально.
